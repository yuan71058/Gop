# OP插件Go语言版本 - 项目说明文档

## 一、项目规划

### 1.1 项目概述
本项目是将原版C++ OP自动化插件转换为Go语言版本。OP插件是一款专为Windows设计的自动化工具，提供屏幕读取、输入模拟和图像处理等功能。

### 1.2 转换目标
- 将C++代码完整转换为Go语言实现
- 保持与原C++版本API的兼容性
- 所有代码添加详细的函数级注释
- 遵循Go语言最佳实践和编码规范

### 1.3 技术栈
- **编程语言**: Go 1.18+
- **目标平台**: Windows (32/64位)
- **依赖**: Windows API (syscall)

## 二、实施方案

### 2.1 项目结构
```
Gop/
├── core/              # 核心类型和工具函数
│   ├── types.go       # 基础类型定义
│   ├── utils.go       # 工具函数
│   └── env.go         # 环境配置
├── winapi/            # Windows API封装
│   └── winapi.go      # 窗口、进程等API
├── background/        # 后台操作模块
│   └── background.go  # 窗口绑定、输入模拟
├── imageproc/         # 图像处理模块
│   └── imageproc.go   # 找图、找色等功能
├── ocr/               # OCR模块
│   └── ocr.go         # 文字识别功能
├── algorithm/         # 算法模块
│   └── astar.go       # A*寻路算法
├── libop/             # 主接口
│   └── libop.go       # OP核心接口类
├── example/           # 示例程序
│   └── main.go        # 使用示例
├── go.mod             # Go模块文件
└── README.md          # 项目说明
```

### 2.2 模块说明

#### 2.2.1 core模块
- **types.go**: 定义核心数据类型 (Point, Rect, Color, OcrRecResult等)
- **utils.go**: 提供字符串处理、键码映射等工具函数
- **env.go**: 环境变量和全局配置管理

#### 2.2.2 winapi模块
- 封装Windows API调用
- 提供窗口查找、位置获取、进程管理等基础功能

#### 2.2.3 background模块
- 实现后台窗口绑定
- 提供鼠标、键盘输入模拟功能
- 支持多种输入模式 (Normal, Windows, DX, DX2)

#### 2.2.4 imageproc模块
- 图像处理核心功能
- 支持找图、找色、多点找色等
- 支持图片缓存机制

#### 2.2.5 ocr模块
- OCR文字识别功能
- 支持HTTP调用Tesseract/PaddleOCR服务
- 提供字典匹配功能

#### 2.2.6 algorithm模块
- A*寻路算法实现
- 用于游戏中的自动寻路等场景

#### 2.2.7 libop模块
- 主接口类，整合所有模块
- 提供与原C++版本一致的API

## 四、使用说明

### 4.1 环境要求
- Go 1.18 或更高版本
- Windows 系统
- GCC/MinGW (如需编译DLL)

### 4.2 编译步骤
```bash
# 进入项目目录
cd E:\SRC\op-master\Gop

# 下载依赖
go mod tidy

# 编译示例程序
go build -o example.exe ./example

# 编译为DLL (可选)
go build -buildmode=c-shared -o gop.dll ./libop
```

### 4.3 使用示例
参考 `example/main.go` 文件
